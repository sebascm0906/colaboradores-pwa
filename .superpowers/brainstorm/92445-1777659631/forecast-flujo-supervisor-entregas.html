Flujo forecast → carga de unidades → visibilidad ruta

Mapa basado en el código actual de la PWA. Verde = ya existe, ámbar = existe parcialmente, rojo = punto que falta amarrar.

1. Supervisor de ventas
Pronostico para manana

Captura fecha objetivo automaticamente como manana.

Alcance

Sucursal completa o vendedor especifico.

Lineas

Producto + canal Van/Mostrador + cantidad.

Confirmar forecast

Llama /gf/salesops/supervisor/v2/forecast/confirm. Aqui debe nacer o actualizar el plan de ruta de manana.

2. Almacenista entregas
Cargar unidades

Consulta rutas de hoy y manana en /pwa-entregas/today-routes.

Bloque por unidad/ruta

Cada tarjeta muestra ruta, chofer, estado, paradas, detalle de carga y stock CEDIS.

Editar carga

Puede ajustar lineas del picking antes de ejecutar la carga.

Confirmar carga

Ejecuta stock CEDIS → unidad via /gf/salesops/warehouse/load/execute. No necesariamente inicia la ruta.

3. App ventas v2 / ruta
Visibilidad esperada

Al confirmar almacenista, la ruta debe quedar visible como publicada/iniciada en ventas.

Hoy la PWA ruta filtra solo hoy

/pwa-ruta/my-plan no toma manana para evitar contaminar la operacion del dia actual.

Transicion critica

Definir si el almacenista solo deja ruta publicada para manana o tambien dispara equivalente de "iniciar/publicar" en Odoo/app ventas.

Cadena tecnica actual

forecast-create

Crea/actualiza gf.saleops.forecast con date_target=manana.

→
forecast-confirm

Backend debe materializar plan/picking para almacen.

→
today-routes

Lista planes de hoy + manana por almacen de despacho.

→
load-execute

Mueve inventario CEDIS → unidad usando el picking del plan.

Decision que falta

A

Dejar lista para manana

El almacenista ejecuta la carga hoy; la ruta queda publicada para manana, pero el vendedor la ve/arranca cuando sea el dia de ruta.

B

Publicar e iniciar al confirmar carga

Al terminar el almacenista, backend replica el efecto operativo de publicar/iniciar ruta en Odoo/app ventas v2.
